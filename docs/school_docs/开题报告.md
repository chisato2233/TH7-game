# 毕业设计（论文）开题报告

## 课题名称

**中文**：基于Unity引擎的多文明策略游戏设计与实现

**英文**：Design and Implementation of Multi-Civilization Strategy Game Based on Unity Engine

---

## 一、选题的背景和意义

### 1.1 研究背景

随着游戏产业的蓬勃发展，策略类游戏作为游戏市场的重要组成部分，以其深度的策略性、丰富的内容体系和较长的游戏生命周期，持续吸引着大量玩家群体。根据中国音数协游戏工委发布的《2024年中国游戏产业报告》，中国游戏市场实际销售收入已突破3000亿元，其中策略类游戏占据重要市场份额。国际上，《文明》系列、《全面战争》系列等经典策略游戏长期保持着极高的玩家活跃度，证明了该类型游戏的市场潜力。

当前市场上的多文明策略游戏大多以欧洲中世纪或现代历史为背景，对亚洲、非洲、大洋洲等地区的文明表现相对不足。《泰拉诺斯大陆：七境英雄》（TH7）项目旨在打破这一局面，以七大独特文明（阿拉伯、埃及、印度、希腊、汉唐、蒙古、南岛）为核心，融合各文明的神话传说、历史英雄和文化特色，构建一个多元化的奇幻世界。

Unity引擎作为全球最流行的游戏开发引擎之一，凭借其跨平台能力、丰富的生态系统和完善的开发工具链，成为独立游戏和中小型游戏开发的首选。Unity 6版本引入了更强大的2D渲染管线（URP）、性能优化工具和现代化开发特性，为本项目的开发提供了坚实的技术基础。

### 1.2 研究意义

**理论意义**：本课题涉及游戏软件架构设计、设计模式应用、事件驱动编程、状态管理等多个软件工程核心领域。通过设计和实现一个完整的策略游戏系统，可以深入理解和实践面向对象设计原则、模块化架构思想，以及游戏开发中的常用设计模式（如单例模式、观察者模式、状态模式、工厂模式等），提升软件工程理论的应用能力。

**实践意义**：本项目的实现将产出一套可复用的游戏开发框架，包括系统管理器、事件系统、上下文管理系统、地图系统等核心模块。这套框架采用分层架构设计，具有良好的可扩展性和可维护性，可为后续游戏开发项目提供参考。同时，通过完整的开发实践，积累从需求分析、架构设计、编码实现到测试优化的全流程项目经验。

**文化意义**：通过游戏这一载体，展现世界多元文明的独特魅力，让玩家在娱乐中了解不同文明的历史文化、神话传说和英雄人物，促进文化传播与交流。

## 二、研究的基本内容和拟解决的主要问题

### 2.1 研究的基本内容

本课题的研究内容主要包括以下几个方面：

**（1）游戏框架架构设计**

设计并实现模块化的游戏系统架构，核心包括：
- **GameEntry系统管理器**：作为游戏入口单例，负责管理所有游戏子系统的生命周期
- **EventSystem事件系统**：基于发布-订阅模式，实现系统间的解耦通信
- **ContextSystem上下文系统**：实现分层上下文管理，支持游戏状态的嵌套、并行和暂停/恢复
- **场景控制器**：管理场景之间的切换和数据传递

**（2）文明系统设计与实现**

实现七大文明的差异化设计体系：
- 文明属性系统：包括主神、文明加成、特色单位等
- 英雄系统：每文明20位特色英雄，包含属性、技能和特殊能力
- 兵种系统：5级兵种体系，涵盖基础单位到神话生物
- 魔法系统：各文明特色法术及跨文明融合魔法

**（3）地图与战斗系统实现**

- 地图系统：采用Unity Tilemap进行可视化编辑，运行时逻辑数据分离
- 地块分层设计：基础地表、地表覆盖、视觉风格三层结构
- 战斗系统：回合制策略战斗机制，兵种克制关系，英雄技能系统

**（4）数据驱动架构**

- 设计可扩展的配置表系统，支持Excel导出和运行时加载
- 采用ScriptableObject进行Unity资源配置
- 实现数据与逻辑的分离，便于数值调整和内容扩展

**（5）用户界面开发**

- 启动界面与主菜单系统
- 世界地图界面与交互
- 战斗界面与战斗反馈
- 文明管理界面（英雄、兵种、魔法等）

### 2.2 拟解决的主要问题

**（1）复杂游戏系统的架构设计问题**

策略游戏涉及大量相互关联的子系统，如何设计一个松耦合、高内聚、易扩展的系统架构是核心挑战。本课题采用分层上下文架构，通过Context树状结构管理游戏状态，支持状态的嵌套（如探索中进入战斗）、并行（多个子系统同时运行）和生命周期管理。

**（2）多文明差异化设计的平衡问题**

七大文明需要在保持各自特色的同时，确保游戏整体的数值平衡。拟通过模块化的文明配置系统，将文明特性数据化，便于调整和测试。同时建立数值平衡测试框架，通过自动化测试验证文明间的相对强度。

**（3）游戏数据的管理与扩展问题**

游戏涉及大量数据（140位英雄、140种兵种、350个事件等），如何有效管理和扩展这些数据是重要问题。拟采用数据驱动架构，将游戏数据与代码逻辑分离，通过配置表（Excel/CSV）和ScriptableObject进行数据管理，支持热更新和模组扩展。

**（4）游戏性能优化问题**

大规模策略游戏需要处理大量单位、复杂的AI决策和丰富的视觉效果，性能优化至关重要。拟采用对象池、延迟加载、分帧处理等技术手段，确保游戏在目标平台上的流畅运行。

## 三、研究方法及措施

### 3.1 研究方法

**（1）文献研究法**

系统学习游戏设计理论、软件架构设计、设计模式等相关文献，重点研究：
- Unity游戏开发技术文档和最佳实践
- 经典游戏设计著作（《游戏编程模式》《游戏设计艺术》等）
- 策略游戏设计案例分析（《文明》《全面战争》等）

**（2）原型迭代法**

采用敏捷开发方法，通过快速原型迭代验证设计方案：
- 快速构建最小可行产品（MVP），验证核心玩法
- 根据测试反馈持续迭代优化
- 采用版本控制（Git）管理开发进度

**（3）模块化设计法**

将复杂系统分解为独立模块，分别设计、实现和测试：
- 定义清晰的模块接口和职责边界
- 采用依赖注入减少模块耦合
- 通过单元测试保证模块质量

**（4）数据驱动设计法**

将游戏内容与程序逻辑分离：
- 使用Excel/CSV管理游戏数值配置
- 采用ScriptableObject存储Unity资源配置
- 实现运行时配置热加载

### 3.2 技术措施

**（1）开发环境与工具**

- 游戏引擎：Unity 6000（LTS版本），URP 2D渲染管线
- 编程语言：C# 12，充分利用现代语言特性
- 版本控制：Git + GitHub
- IDE：Visual Studio 2022 / Rider
- 美术工具：Aseprite（像素美术）、Photoshop（UI设计）

**（2）核心技术方案**

- 采用单例模式实现GameEntry系统管理器
- 采用观察者模式实现EventSystem事件系统
- 采用状态模式管理游戏上下文切换
- 采用Unity Tilemap + 逻辑层分离实现地图系统
- 采用DOTween实现动画效果
- 采用UniTask处理异步操作

**（3）质量保证措施**

- 编写单元测试覆盖核心逻辑模块
- 使用Unity Profiler进行性能分析优化
- 建立代码规范并严格执行
- 定期进行代码审查和重构
